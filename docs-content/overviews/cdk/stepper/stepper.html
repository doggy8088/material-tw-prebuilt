CDK 步進器（stepper）提供了一個基礎，可以在此基礎上建立具體的步進器變體。步進器是一種類似嚮導的工作流，可以把內容劃分為一些邏輯步驟

CDK stepper provides a foundation upon which more concrete stepper varieties can be built. A stepper is a wizard-like workflow that divides content into logical steps

CdkStepper 所捕獲的行為

Behavior captured by CdkStepper

步進器的 CDK 基礎版本主要用來管理哪一步處於活動狀態。這包括處理鍵盤互動，並暴露一個用來在工作流程中前進或後退的 API。

The base CDK version of the stepper primarily manages which step is active. This includes handling keyboard interactions and exposing an API for advancing or rewinding through the workflow.

線性步進器

Linear stepper

標記為 linear 的步進器要求使用者在繼續下一步之前完成前面的步驟。對於每一個步驟來說，其 stepControl 屬性都可以設定為頂級的 AbstractControl，用於檢查該步驟的有效性。

A stepper marked as linear requires the user to complete previous steps before proceeding. For each step, the stepControl attribute can be set to the top level AbstractControl that is used to check the validity of the step.

有兩種可能的方式。一種是使用單一表單進行步進，另一種是使用不同的表單進行步進。

There are two possible approaches. One is using a single form for stepper, and the other is using a different form for each step.

或者，如果你不想使用 Angular 表單，也可以把 completed 屬性傳入每個步驟，這些步驟會阻止使用者繼續，直到這個屬性變為 true。注意，如果同時設定了 completed 和 stepControl，則 stepControl 優先。

Alternatively, if you don't want to use the Angular forms, you can pass in the completed property to each of the steps which won't allow the user to continue until it becomes true. Note that if both completed and stepControl are set, the stepControl will take precedence.

在整個步進器中使用同一個表單

Using a single form for the entire stepper

在步進器中使用單一表單時，所有中間步驟的"下一個"/"上一個"按鈕都必須設定為 type="button"，以防止在完成所有步驟之前送出表單。

When using a single form for the stepper, any intermediate next/previous buttons within the steps must be set to type="button" in order to prevent submission of the form before all steps are complete.

每個步驟各自使用一個表單

Using a form for each individual step

當每個步驟各自使用一個表單時，只要送出了一個表單，工作流就會向前推進。

When using a form for each step, the workflow is advanced whenever one of the forms is submitted.

步驟的型別

Types of steps

可選步驟

Optional step

如果線性步進器中的不要求某個步驟必須完成，那麼可以在 linear 步進器中的這個 CdkStep 上設定 optional 屬性。

If completion of a step in linear stepper is not required, then the optional attribute can be set on CdkStep in a linear stepper.

可編輯的步驟

Editable step

預設情況下，步驟是可編輯的，這意味著使用者可以回傳之前已完成的步驟並重新編輯他們的回應。可以在 CdkStep 上設定 editable="true" 來修改預設行為。

By default, steps are editable, which means users can return to previously completed steps and edit their responses. editable="false" can be set on CdkStep to change the default.

完成步驟

Completed step

預設情況下，如果步驟有效（線上性步進器的情況下）並且使用者已經與該步驟互動過，則該步驟的 completed 屬性會回傳 true。但是，使用者也可以按需設定 completed 屬性來覆蓋預設的 completed 行為。

By default, the completed attribute of a step returns true if the step is valid (in case of linear stepper) and the user has interacted with the step. The user, however, can also override this default completed behavior by setting the completed attribute as needed.

步進按鈕

Stepper buttons

有兩個按鈕指令可以支援不同步驟之間的導向： CdkStepperNext 和 CdkStepperPrevious。當放置在某個步驟中時，它們會自動新增點選處理器來分別推進或回退此工作流。

There are two button directives to support navigation between different steps: CdkStepperNext and CdkStepperPrevious. When placed inside a step, these will automatically add click handlers to advance or rewind the workflow, respectively.

重置步進器的某個步驟

Resetting a stepper

如果要將步進器重置為初始狀態，可以使用 reset 方法。注意，重置它會在底部操作表控制元件上呼叫 reset 來清除這個值。

If you want to reset a stepper to its initial state, you can use the reset method. Note that resetting it will call reset on the underlying form control which clears the value.

無障礙性

Accessibility

除了內建的鍵盤支援外，步進器不進行任何處理。在實現你自己的元件時，建議將步進器視為無障礙性的頁籤式檢視，方法是為其指定 role="tablist"。可以點選具有 role="tab" 屬性的步驟頭部來選擇此步驟，選擇後能展開的內容應該具有 role="tabpanel" 屬性。此外，步驟標題應該有一個 aria-selected 屬性來反映它的選擇狀態，並且相關的內容元素應該有 aria-expanded 屬性。

Apart from the built-in keyboard support, the stepper doesn't apply any treatment. When implementing your own component, it is recommended that the stepper is treated as a tabbed view for accessibility purposes by giving it a role="tablist". The header of step that can be clicked to select the step should be given role="tab", and the content that can be expanded upon selection should be given role="tabpanel". Furthermore, the step header should have an aria-selected attribute that reflects its selected state.

你可以參考 Angular Material 步進器作為無障礙實現的範例。

You can refer to the Angular Material stepper as an example of an accessible implementation.
