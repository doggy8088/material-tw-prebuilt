Angular Material 按鈕就是原生的 <button> 或 <a> 元素，但使用 Material Design 的樣式和墨水漣漪等效果進行了增強。

Angular Material buttons are native <button> or <a> elements enhanced with Material Design styling and ink ripples.

原生的 <button> 和 <a> 元素經常用來為使用者提供最直白、易用的互動體驗。 當要執行某些動作時，應該使用 <button> 元素。 當用戶要導向到其它檢視時，應該使用 <a> 元素。

Native <button> and <a> elements are always used in order to provide the most straightforward and accessible experience for users. A <button> element should be used whenever some action is performed. An <a> element should be used whenever the user will navigate to another view.

主題

Theming

可以透過把 color 屬性設定為 primary、accent 或 warn 來把按鈕的背景色設定當前主題中的顏色。

Buttons can be colored in terms of the current theme using the color property to set the background color to primary, accent, or warn.

大寫化

Capitalization

根據 Material Design 規範，按鈕文字應該是大寫的，不過我們選擇不讓它自動透過 text-transform: uppercase 進行大寫化，因為這可能會在某些語言環境下導致問題。 值得注意的是，如果把文字本身寫成全大寫形式（而不是透過 css），可能會導致某些螢幕閱讀器出現問題 —— 它將會一個字元一個字元的把文字讀出來。 所以我們決定讓應用開發者自己來決定用哪種形式。

According to the Material design spec button text has to be capitalized, however we have opted not to capitalize buttons automatically via text-transform: uppercase, because it can cause issues in certain locales. It is also worth noting that using ALL CAPS in the text itself causes issues for screen-readers, which will read the text character-by-character. We leave the decision of how to approach this to the consuming app.

一些擴充套件 Fab 按鈕

Extended fab buttons

傳統的 FAB 按鈕是原型的，並且只有一個單個圖示的空間，你可以新增 extended 屬性，讓此 Fab 能擴充套件成圓角矩形，以便在圖示之外，擴充套件出容納文字標籤的空間。只有完全打消的 Fab 才支援 extended 屬性，微型（mini） Fab 不支援。

Traditional fab buttons are circular and only have space for a single icon. However, you can add the extended attribute to allow the fab to expand into a rounded rectangle shape with space for a text label in addition to the icon. Only full sized fabs support the extended attribute, mini fabs do not.

<button mat-fab extended>
  <mat-icon>home</mat-icon>
  Home
</button>

無障礙性

Accessibility

Angular Material 使用原生的 <button> 和 <a> 元素，來保留預設的無障礙性體驗。 當要執行某些動作時，應該使用 <button> 元素。 當用戶要導向到其它檢視時，應該使用 <a> 元素。

Angular Material uses native <button> and <a> elements to ensure an accessible experience by default. A <button> element should be used for any interaction that performs an action on the current page. An <a> element should be used for any interaction that navigates to another URL. All standard accessibility best practices for buttons and anchors apply to MatButton.

停用錨點

Disabling anchors

除了原生 <a> 元素提供的屬性外， MatAnchor 還支援停用錨點。當你停用錨點時，元件會設定 aria-disabled="true" 和 tabindex="-1" 。始終在你的應用程式中測試停用的錨點，以確保與你的應用程式支援的任何輔助技術相容。

MatAnchor supports disabling an anchor in addition to the features provided by the native <a> element. When you disable an anchor, the component sets aria-disabled="true" and tabindex="-1". Always test disabled anchors in your application to ensure compatibility with any assistive technology your application supports.

帶圖示的按鈕

Buttons with icons

應該透過 aria-label 或 aria-labelledby 為僅包含圖示的按鈕或連結（例如 mat-fab 、 mat-mini-fab 和 mat-icon-button ）賦予一個有意義的標籤。有關在按鈕中使用圖示的更多訊息，請參閱 MatIcon 的文件。

Buttons or links containing only icons (such as mat-fab, mat-mini-fab, and mat-icon-button) should be given a meaningful label via aria-label or aria-labelledby. See the documentation for MatIcon for more information on using icons in buttons.

切換按鈕

Toggle buttons

對於那些只含圖示的按鈕或連結（比如 mat-fab、mat-mini-fab 和 mat-icon-button），應該透過 aria-label 或 aria-labelledby 給出一個有意義的標籤。

See the documentation for MatButtonToggle for information on stateful toggle buttons.
