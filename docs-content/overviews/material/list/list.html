<mat-list> 是一個容器組件，用於包裝和格式化一系列條目。 作為最基本的清單元件，它提供了 Material Design 的樣式，但是並沒有定義屬於自己的行為。

<mat-list> is a container component that wraps and formats a series of <mat-list-item>. As the base list component, it provides Material Design styling, but no behavior of its own.

清單條目可以透過兩種方式建立，具體取決於它們需要顯示的內容：

List items can be constructed in two ways depending the content they need to show:

簡單清單

Simple lists

如果清單條目需要顯示單行文字訊息，可以將文字直接插入到 <mat-list-item> 元素中。

If a list item needs to show a single line of textual information, the text can be inserted directly into the <mat-list-item> element.

<mat-list>
 <mat-list-item>Pepper</mat-list-item>
 <mat-list-item>Salt</mat-list-item>
 <mat-list-item>Paprika</mat-list-item>
</mat-list>

多行清單

Multi-line lists

具有多行文字的清單條目必須使用 matListItemTitle 指令來指示其標題文字以實現無障礙性目的，但 matListItemLine 指令除外，它用於每個後續文字行。

List items that have more than one line of text have to use the matListItemTitle directive to indicate their title text for accessibility purposes, in addition to the matListItemLine directive for each subsequent line of text.

<mat-list>
  <mat-list-item>
    <span matListItemTitle>Pepper</span>
    <span matListItemLine>Produced by a plant</span>
  </mat-list-item>
  <mat-list-item>
    <span matListItemTitle>Salt</span>
    <span matListItemLine>Extracted from sea water</span>
  </mat-list-item>
  <mat-list-item>
    <span matListItemTitle>Paprika</span>
    <span matListItemLine>Produced by dried and ground red peppers</span>
  </mat-list-item>
</mat-list>

要啟用文字換行，必須在 <mat-list-item> 上設定 lines 輸入以指示文字行數。

To activate text wrapping, the lines input has to be set on the <mat-list-item> indicating the number of lines of text.

以下指令可用於設定清單條目內容的樣式：

The following directives can be used to style the content of a list item:

指令Directive 說明Description
matListItemTitle 指示清單條目的標題。多行清單條目需要。Indicates the title of the list item. Required for multi-line list items.
matListItemLine 在清單條目中進行文字換行。Wraps a line of text within a list item.
matListItemIcon 圖示通常放置在清單條目的開頭。Icon typically placed at the beginning of a list item.
matListItemAvatar 影象通常放置在清單條目的開頭。Image typically placed at the beginning of a list item.
matListItemMeta 在清單條目末尾的 meta 部分中插入內容。Inserts content in the meta section at the end of a list item.

導向清單

Navigation lists

使用 mat-nav-list 標記來表示導向清單（即帶連結的清單）。

Use mat-nav-list tags for navigation lists (i.e. lists that have anchor tags).

簡單導向清單可以直接把 mat-list-item 屬性用在連結上：

Simple navigation lists can use the mat-list-item attribute on anchor tag elements directly:

<mat-nav-list>
  <a mat-list-item href="..." *ngFor="let link of links" [activated]="link.isActive">{{ link }}</a>
</mat-nav-list>

對於更復雜的導向清單（比如：每個條目具有多個目標時），可以把連結包裹在 <mat-list-item> 中。

For more complex navigation lists (e.g. with more than one target per item), wrap the anchor element in an <mat-list-item>.

<mat-nav-list>
  <mat-list-item *ngFor="let link of links" [activated]="link.isActive">
     <a matListItemTitle href="...">{{ link }}</a>
     <button mat-icon-button (click)="showInfo(link)" matListItemMeta>
        <mat-icon>info</mat-icon>
     </button>
  </mat-list-item>
</mat-nav-list>

動作清單

Action lists

當清單中的每個項都要執行某種動作時，使用 <mat-action-list> 元素。動作清單中的每一項都是一個 <button> 元素。

Use the <mat-action-list> element when each item in the list performs some action. Each item in an action list is a <button> element.

簡單的動作清單可以直接用 mat-list-item 屬性新增在 button 元素上。

Simple action lists can use the mat-list-item attribute on button tag elements directly:

<mat-action-list>
  <button mat-list-item (click)="save()">Save</button>
  <button mat-list-item (click)="undo()">Undo</button>
</mat-action-list>

選取清單

Selection lists

選取清單提供了一種可以選取值的介面，清單中的每個條目都是一個選項。

A selection list provides an interface for selecting values, where each list item is an option.

選取清單中的選項不應該包含可互動控制元件，比如按鈕或連結。

The options within a selection-list should not contain further interactive controls, such as buttons and anchors.

多行清單

Multi-line lists

對於每個條目需要多行內容的清單，可以給每一行標註 matLine 屬性。 這裡應該使用一個在你的 DOM 層次下最恰當的標題標記（不一定要要像下面的例子中這樣用 <h3>）。

For lists that require multiple lines per item, annotate each line with an matListItemLine attribute. Whichever heading tag is appropriate for your DOM hierarchy should be used (not necessarily <h3> as shown in the example).

<!-- two line list -->
<mat-list>
  <mat-list-item *ngFor="let message of messages">
    <h3 matListItemTitle>{{message.from}}</h3>
    <p matListItemLine>
      <span>{{message.subject}}</span>
      <span class="demo-2"> -- {{message.content}}</span>
    </p>
  </mat-list-item>
</mat-list>

<!-- three line list -->
<mat-list>
  <mat-list-item *ngFor="let message of messages">
    <h3 matListItemTitle>{{message.from}}</h3>
    <p matListItemLine>{{message.subject}}</p>
    <p matListItemLine class="demo-2">{{message.content}}</p>
  </mat-list-item>
</mat-list>

帶圖示的清單

Lists with icons

要想給清單條目新增圖示，請使用 matListIcon 屬性。

To add an icon to your list item, use the matListItemIcon attribute.

<mat-list>
  <mat-list-item *ngFor="let message of messages">
    <mat-icon matListItemIcon>folder</mat-icon>
    <h3 matListItemTitle>{{message.from}}</h3>
    <p matListItemLine>
      <span>{{message.subject}}</span>
      <span class="demo-2"> -- {{message.content}}</span>
    </p>
  </mat-list-item>
</mat-list>

帶頭像的清單

Lists with avatars

要包含一個頭像，請新增一個帶有 matListAvatar 屬性的影象標記。

To include an avatar image, add an image tag with an matListItemAvatar attribute.

<mat-list>
  <mat-list-item *ngFor="let message of messages">
    <img matListItemAvatar src="..." alt="...">
    <h3 matListItemTitle>{{message.from}}</h3>
    <p matListItemLine>
      <span>{{message.subject}}</span>
      <span class="demo-2"> -- {{message.content}}</span>
    </p>
  </mat-list-item>
</mat-list>

帶有多個分割槽的清單

Lists with multiple sections

可以透過帶 matSubheader 屬性的標題標記來為清單新增子標題。 要想新增分隔符，可以用 <mat-divider>。

Subheaders can be added to a list by annotating a heading tag with an matSubheader attribute. To add a divider, use <mat-divider>.

<mat-list>
   <h3 matSubheader>Folders</h3>
   <mat-list-item *ngFor="let folder of folders">
      <mat-icon matListIcon>folder</mat-icon>
      <h4 matListItemTitle>{{folder.name}}</h4>
      <p matListItemLine class="demo-2"> {{folder.updated}} </p>
   </mat-list-item>
   <mat-divider></mat-divider>
   <h3 matSubheader>Notes</h3>
   <mat-list-item *ngFor="let note of notes">
      <mat-icon matListIcon>note</mat-icon>
      <h4 matListItemTitle>{{note.name}}</h4>
      <p matListItemLine class="demo-2"> {{note.updated}} </p>
   </mat-list-item>
</mat-list>

無障礙性

Accessibility

Angular Material 提供了多種清單，以便你可以選擇最適合你的用例的那種。

Angular Material offers multiple varieties of list so that you can choose the type that best applies to your use-case.

導向

Navigation

當清單中的每個條目都是導向到另一個 URL 的錨點時，你應該使用 MatNavList 。根 <mat-nav-list> 元素會設定 role="navigation" 並且應該只包含具有 mat-list-item 屬性的錨點元素。你不應在這些錨點內嵌套任何互動元素，包括按鈕和複選框。

You should use MatNavList when every item in the list is an anchor that navigate to another URL. The root <mat-nav-list> element sets role="navigation" and should contain only anchor elements with the mat-list-item attribute. You should not nest any interactive elements inside these anchors, including buttons and checkboxes.

始終透過 aria-label 或 aria-labelledby 為 <mat-nav-list> 元素提供無障礙標籤。

Always provide an accessible label for the <mat-nav-list> element via aria-label or aria-labelledby.

選取結果

Selection

對於允許使用者選擇一個或多個值的清單，你應該使用 MatSelectionList 和 MatListOption 。此清單變體使用 role="listbox" 互動模式，處理所有相關的鍵盤輸入和焦點管理。你不應在這些選項中巢狀任何互動元素，包括按鈕和錨點。

You should use MatSelectionList and MatListOption for lists that allow the user to select one or more values. This list variant uses the role="listbox" interaction pattern, handling all associated keyboard input and focus management. You should not nest any interactive elements inside these options, including buttons and anchors.

始終透過 aria-label 或 aria-labelledby 為 <mat-selection-list> 元素提供一個無障礙標籤，以描述正在進行的選擇。

Always provide an accessible label for the <mat-selection-list> element via aria-label or aria-labelledby that describes the selection being made.

預設情況下， MatSelectionList 會顯示單選或複選標記指示器來標識所選條目。 雖然你可以透過 hideSingleSelectionIndicator 來隱藏用於單選的單選指示器，但這會使用戶更難甚至不可能直觀地識別所選條目，從而降低元件的無障礙性。

By default, MatSelectionList displays radio or checkmark indicators to identify selected items. While you can hide the radio indicator for single-selection via hideSingleSelectionIndicator, this makes the component less accessible by making it harder or impossible for users to visually identify selected items.

自定義方案

Custom scenarios

預設情況下，清單元件假定自己是純裝飾性的，因此不設定任何角色、ARIA 屬性或鍵盤快捷鍵。 這相當於頁面上有一系列 <div> 元素。清單內部的任何互動式內容都應該根據應用程式的特定工作流進行適當的無障礙性處理。

By default, the list assumes that it will be used in a purely decorative fashion and thus it sets no roles, ARIA attributes, or keyboard shortcuts. This is equivalent to having a sequence of <div> elements on the page. Any interactive content within the list should be given an appropriate accessibility treatment based on the specific workflow of your application.

如果清單元件用於渲染非互動式內容項的清單，那麼清單元素應該帶有 role="list" 屬性，並且每個清單條目都應該帶有 role="listitem" 屬性。

If the list is used to present a list of non-interactive content items, then the list element should be given role="list" and each list item should be given role="listitem".
