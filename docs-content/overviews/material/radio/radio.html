<mat-radio> 提供了與原生元素 <input type="radio"> 相同的功能，但用 Material Design 的樣式和動畫進行了增強。

<mat-radio-button> provides the same functionality as a native <input type="radio"> enhanced with Material Design styling and animations.

所有具有相同 name 的單選按鈕會構成一個集合，一次只能選擇一個。

All radio-buttons with the same name comprise a set from which only one may be selected at a time.

單選按鈕的標籤

Radio-button label

單選按鈕的標籤是透過 <mat-radio-button> 元素的內容提供的。 該標籤可以透過把 labelPosition 屬性的值設定為 'before' 或 'after' 來把該標籤定位在單選按鈕的前面或後面。

The radio-button label is provided as the content to the <mat-radio-button> element. The label can be positioned before or after the radio-button by setting the labelPosition property to 'before' or 'after'.

如果你不想讓此標籤出現在單選按鈕的緊後面，你可以用 aria-label 或 aria-labelledby 來指定一個合適的標籤。

If you don't want the label to appear next to the radio-button, you can use aria-label or aria-labelledby to specify an appropriate label.

單選組

Radio groups

除非 DOM 結構不允許（比如在表格單元格中的單選按鈕），否則單選按鈕通常都要放在 <mat-radio-group> 的內部。 單選組具有一個 value 屬性，用於表示該組中當前選中的單選按鈕。

Radio-buttons should typically be placed inside of an <mat-radio-group> unless the DOM structure would make that impossible (e.g., radio-buttons inside of table cells). The radio-group has a value property that reflects the currently selected radio-button inside of the group.

單選組中的每個單選按鈕都會繼承該組的 name。

Individual radio-buttons inside of a radio-group will inherit the name of the group.

和 @angular/forms 一起使用

Use with @angular/forms

<mat-radio-group> 與 @angular/forms 相容，並且同時支援 FormsModule 和 ReactiveFormsModule。

<mat-radio-group> is compatible with @angular/forms and supports both FormsModule and ReactiveFormsModule.

預設顏色設定

Default Color Configuration

可以使用 MAT_RADIO_DEFAULT_OPTIONS 提供者全域設定單選按鈕的預設顏色

The default color for radio buttons can be configured globally using the MAT_RADIO_DEFAULT_OPTIONS provider

providers: [{
    provide: MAT_RADIO_DEFAULT_OPTIONS,
    useValue: { color: 'accent' },
}]

無障礙性

Accessibility

MatRadioButton 使用內部的 <input type="radio"> 來提供無障礙體驗。這個內部單選按鈕會接收焦點並由 <mat-radio-button> 元素的文字內容自動標記。儘量不要在 <mat-radio-button> 的內容中新增其他可互動控制元件，因為這會降低輔助技術使用者的體驗。

MatRadioButton uses an internal <input type="radio"> to provide an accessible experience. This internal radio button receives focus and is automatically labelled by the text content of the <mat-radio-button> element. Avoid adding other interactive controls into the content of <mat-radio-button>, as this degrades the experience for users of assistive technology.

始終透過 aria-label 或 aria-labelledby 為沒有描述性文字內容的單選按鈕提供無障礙標籤。對於動態標籤和描述， MatRadioButton 提供了用於繫結 aria-label 、 aria-labelledby 和 aria-describedby 的輸入屬性。這意味著你不應該使用 attr. 作為繫結這些屬性時的字首，如下所示。

Always provide an accessible label via aria-label or aria-labelledby for radio buttons without descriptive text content. For dynamic labels and descriptions, MatRadioButton provides input properties for binding aria-label, aria-labelledby, and aria-describedby. This means that you should not use the attr. prefix when binding these properties, as demonstrated below.

<mat-radio-button [aria-label]="getMultipleChoiceAnswer()">
</mat-radio-button>

推薦你將所有單選按鈕放在 <mat-radio-group> 內而不是建立獨立的單選按鈕，因為組更容易用於鍵盤。

Prefer placing all radio buttons inside of a <mat-radio-group> rather than creating standalone radio buttons because groups are easier to use exclusively with a keyboard.

你應該透過 aria-label 或 aria-labelledby 為所有 <mat-radio-group> 元素提供一個無障礙標籤。

You should provide an accessible label for all <mat-radio-group> elements via aria-label or aria-labelledby.
