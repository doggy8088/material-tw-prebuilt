<mat-select> 是一個表單控制元件，像原生的 <select> 元素一樣，它用於從一組選項中選取一個值。 你可以閱讀 Material Design 規範瞭解更多訊息。 它被設計成在 <mat-form-field> 元素內部使用的。

<mat-select> is a form control for selecting a value from a set of options, similar to the native <select> element. You can read more about selects in the Material Design spec. It is designed to work inside of a <mat-form-field> element.

要想往選擇框中新增選項，請為 <mat-select> 新增 <mat-option> 元素。每個 <mat-option> 都有一個 value 屬性，用於給出當用戶選擇了該選項時的結果值，而 <mat-option> 的內容則會顯示給使用者。

To add options to the select, add <mat-option> elements to the <mat-select>. Each <mat-option> has a value property that can be used to set the value that will be selected if the user chooses this option. The content of the <mat-option> is what will be shown to the user.

Angular Material 還支援在 <mat-form-field> 中使用原生的 <select> 元素。 這種原生控制元件在效能、無障礙性、可用性等方面具有一系列優點。參見 form-field 的文件以瞭解詳情。

Angular Material also supports use of the native <select> element inside of <mat-form-field>. The native control has several performance, accessibility, and usability advantages. See the documentation for form-field for more information.

要在 <mat-form-field> 中使用原生 select，請為 <select> 元素新增 matNativeControl 屬性。

To use a native select inside <mat-form-field>, import MatInputModule and add the matNativeControl attribute to the <select> element.

獲取與設定選擇框的值

Getting and setting the select value

<mat-select> 支援對 value 屬性進行雙向繫結，而不需要藉助 Angular 的表單支援。

The <mat-select> supports 2-way binding to the value property without the need for Angular forms.

<mat-select> 也支援所有的表單指令，包括來自 FormsModule 的 NgModel 和來自 ReactiveFormsModule 的 FormControl、FormGroup 等。像原生的 <select> 一樣，<mat-select> 也支援 compareWith 函式。（要了解自定義 compareWith 的方法，參見 Angular 表單文件。）

Both<mat-select> and <select> support all of the form directives from the core FormsModule (NgModel) and ReactiveFormsModule (FormControl, FormGroup, etc.) As with native <select>, <mat-select> also supports a compareWith function. (Additional information about using a custom compareWith function can be found in the Angular forms documentation).

表單欄位的屬性

Form field features

<mat-form-field> 的幾個屬性可以和任何 <mat-select> 一起使用。包括：錯誤訊息、提示訊息、字首與字尾、主題。 欲知詳情，參見表單欄位的文件。

There are a number of <mat-form-field> features that can be used with both <select> and <mat-select>. These include error messages, hint text, prefix & suffix, and theming. For additional information about these features, see the form field documentation.

設定靜態佔位符

Setting a static placeholder

佔位符是指當 <mat-form-field> 浮起，但 <mat-select> 為空時顯示的文字。 它可以給使用者提供一些關於如何進行選擇的額外訊息。 佔位符可以透過 <mat-select> 元素上的 placeholder 屬性來指定。 有些情況下，<mat-form-field> 可以把佔位符用作標籤（參見表單欄位文件）。

The placeholder is text shown when the <mat-form-field> label is floating but the <mat-select> is empty. It is used to give the user an additional hint about the value they should select. The placeholder can be specified by setting the placeholder attribute on the <mat-select> element. In some cases that <mat-form-field> may use the placeholder as the label (see the form field label documentation).

停用選擇框或其中的某個選項

Disabling the select or individual options

可以透過 <mat-select> 上的 disabled 屬性停用整個選擇框，或透過 <mat-option> 上的 disabled 屬性單獨停用某一個選項。 當在回應式表單中使用時，選擇框元件可以透過表單控制元件進行啟用/停用。 這可以透過建立一個帶有 disabled 屬性的 FormControl（FormControl({value: '', disabled: true})）來完成，也可以使用 FormControl.enable() 或 FormControl.disable()

It is possible to disable the entire select or individual options in the select by using the disabled property on the <select> or <mat-select> and the <option> or <mat-option> elements respectively. When working with Reactive Forms, the select component can be disabled/enabled via form controls. This can be accomplished by creating a FormControl with the disabled property FormControl({value: '', disabled: true}) or using FormControl.enable(), FormControl.disable().

重置選擇框的值

Resetting the select value

如果你希望用某個選項重置選擇框的值，那麼可以省略它的值。

If you want one of your options to reset the select's value, you can omit specifying its value.

建立選項組

Creating groups of options

<mat-optgroup> 元素可以用來把一些選項分組到子標題下。組的名字可以用 <mat-optgroup> 的 label 來指定。 像 <mat-option> 元素一樣，<mat-optgroup> 也能透過設定組的 disabled 屬性來停用或啟用。

The <mat-optgroup> element can be used to group common options under a subheading. The name of the group can be set using the label property of <mat-optgroup>. Like individual <mat-option> elements, an entire <mat-optgroup> can be disabled or enabled by setting the disabled property on the group.

多選

Multiple selection

<mat-select> 預設為單選模式，但是可以透過 multiple 屬性設定成允許多選的，這樣使用者就可以同時選擇多個值了。 當在多選模式下使用 <mat-select> 時，它的值將會是所有選定值的有序清單，而不再是單個值。

<mat-select> defaults to single-selection mode, but can be configured to allow multiple selection by setting the multiple property. This will allow the user to select multiple values at once. When using the <mat-select> in multiple selection mode, its value will be a sorted list of all selected values rather than a single value.

不鼓勵在 <mat-form-field> 中使用原生選擇框（<select multiple>）來實現多選功能，因為行內的多項選擇框在外觀上會與其它 Material 元件不一致。

Using multiple selection with a native select element (<select multiple>) is discouraged inside <mat-form-field>, as the inline listbox appearance is inconsistent with other Material Design components.

定製觸發器標籤

Customizing the trigger label

如果你想在選擇框內部顯示一個自定義的觸發器標籤，可以使用 <mat-select-trigger> 元素。

If you want to display a custom trigger label inside a <mat-select>, you can use the <mat-select-trigger> element.

停用漣漪效果

Disabling the ripple effect

預設情況下，當用戶點選某個 <mat-option> 時，就會顯示一個漣漪動畫。可以透過設定 <mat-select> 上的 disableRipple 屬性來停用它。

By default, when a user clicks on a <mat-option>, a ripple animation is shown. This can be disabled by setting the disableRipple property on <mat-select>.

為下拉面板新增自定義樣式

Adding custom styles to the dropdown panel

為了便於給下拉面板指定樣式，<mat-select> 有一個 panelClass 屬性，可用於為下拉面板指定自定義 CSS 類。

In order to facilitate easily styling the dropdown panel, <mat-select> has a panelClass property which can be used to apply additional CSS classes to the dropdown panel.

更改錯誤訊息的顯示時機

Changing when error messages are shown

<mat-form-field> 讓你能把錯誤訊息和你的 <mat-select> 關聯起來。 預設情況下，當該控制元件無效並且使用者已經跟它互動過（碰過 touched）或者送出過父表單時才會顯示這些錯誤訊息。 如果你要覆蓋這種行為（比如當無效控制元件或父表單組變髒時想盡快顯示錯誤訊息），可以使用 <mat-select> 的 errorStateMatcher 屬性。 該屬性接收一個 ErrorStateMatcher 物件的實例。ErrorStateMatcher 必須實現一個 isErrorState 方法，它接受該 <mat-select> 或其父表單的 FormControl，並回傳一個 boolean 值，以指出是否應該顯示錯誤訊息。（true 表示應該顯示，false 表示不該。）

The <mat-form-field> allows you to associate error messages with your <select> or <mat-select>. By default, these error messages are shown when the control is invalid and either the user has interacted with (touched) the element or the parent form has been submitted. If you wish to override this behavior (e.g. to show the error as soon as the invalid control is dirty or when a parent form group is invalid), you can use the errorStateMatcher property of the <mat-select>. The property takes an instance of an ErrorStateMatcher object. An ErrorStateMatcher must implement a single method isErrorState which takes the FormControl for this <mat-select> as well as the parent form and returns a boolean indicating whether errors should be shown. (true indicating that they should be shown, and false indicating that they should not.)

全域的錯誤狀態對應器可以透過 ErrorStateMatcher 提供者來指定。它將作用於所有輸入框。 為方便起見，可以使用 ShowOnDirtyErrorStateMatcher 作為全域設定，來要求只有當輸入框是髒（dirty）且無效（invalid）時才顯示錯誤訊息。

A global error state matcher can be specified by setting the ErrorStateMatcher provider. This applies to all inputs. For convenience, ShowOnDirtyErrorStateMatcher is available in order to globally cause input errors to show when the input is dirty and invalid.

@NgModule({
  providers: [
    {provide: ErrorStateMatcher, useClass: ShowOnDirtyErrorStateMatcher}
  ]
})

無障礙性

Accessibility

如果可能，更推薦使用原生 <select> 元素而不是 MatSelect 。原生控制元件能在最廣泛的平台上提供無障礙體驗。

When possible, prefer a native <select> element over MatSelect. The native control provides the most accessible experience across the widest range of platforms.

MatSelect 實現了 ARIA 規範 1.2 版中詳述的組合框模式。組合框觸發器會控制在彈出視窗中開啟的 role="listbox" 元素。以前版本的 ARIA 規範要求 role="combobox" 應用於文字輸入控制元件，但規範的 1.2 版本支援更廣泛的互動模式。 ARIA 的這種新用法適用於 Angular Material 支援的所有瀏覽器和螢幕閱讀器組合。

MatSelect implements the combobox pattern detailed in the 1.2 version of the ARIA specification. The combobox trigger controls a role="listbox" element opened in a pop-up. Previous versions of the ARIA specification required that role="combobox" apply to a text input control, but the 1.2 version of the specification supports a wider variety of interaction patterns. This newer usage of ARIA works in all browser and screen-reader combinations supported by Angular Material.

由於彈出視窗使用 role="listbox" 模式，因此不應將其他互動控制元件（例如按鈕或複選框）放入選擇選項中。如果這樣巢狀可互動控制元件會干擾大多數輔助技術。

Because the pop-up uses the role="listbox" pattern, you should not put other interactive controls, such as buttons or checkboxes, inside a select option. Nesting interactive controls like this interferes with most assistive technology.

始終要為選擇框提供一個無障礙標籤。這可以透過在 <mat-form-field> 中新增一個 <mat-label> 元件或 aria-label、aria-labelledby 屬性來實現。

Always provide an accessible label for the select. This can be done by adding a <mat-label> inside of <mat-form-field>, the aria-label attribute, or the aria-labelledby attribute.

預設情況下， MatSelect 會顯示一個選中標記來標識所選條目。 雖然你可以透過 hideSingleSelectionIndicator 來隱藏單選時的選中標記指示器，但這會使用戶更難甚至不可能直觀地識別所選條目，從而降低元件的無障礙性。

By default, MatSelect displays a checkmark to identify selected items. While you can hide the checkmark indicator for single-selection via hideSingleSelectionIndicator, this makes the component less accessible by making it harder or impossible for users to visually identify selected items.

排查問題

Troubleshooting

Error: Cannot change multiple mode of select after initialization
（不能在選擇框初始化完成後修改 multiple 模式）

Error: Cannot change multiple mode of select after initialization

如果你試圖把 <mat-select> 的 multiple 屬性繫結到一個動態值，就會丟擲此錯誤。（比如 [multiple]="isMultiple" 如果 isMultiple 的值在元件生命週期內發生了變化。）如果你確實需要動態修改它，請改用 ngIf 或 ngSwitch：

This error is thrown if you attempt to bind the multiple property on <mat-select> to a dynamic value. (e.g. [multiple]="isMultiple" where the value of isMultiple changes over the course of the component's lifetime). If you need to change this dynamically, use ngIf or ngSwitch instead:

<mat-select *ngIf="isMultiple" multiple>
  ...
</mat-select>
<mat-select *ngIf="!isMultiple">
  ...
</mat-select>

Error: Value must be an array in multiple-selection mode
（在多選模式下，值必須是陣列）

Error: Value must be an array in multiple-selection mode

如果你試圖給 <mat-select multiple> 賦值為除 null、undefined 或陣列之外的值，就會丟擲此錯誤。 比如，mySelect.value = 'option1' 就會出錯，應該寫成 mySelect.value = ['option1']。

This error is thrown if you attempt to assign a value other than null, undefined, or an array to a <mat-select multiple>. For example, something like mySelect.value = 'option1'. What you likely meant to do was mySelect.value = ['option1'].

Error: compareWith must be a function
（compareWith 必須是函式）

Error: compareWith must be a function

如果你嘗試把函式之外的值賦給 compareWith 屬性，就會出現此錯誤。要了解如何正確使用 compareWith，請參見 Angular 表單文件。

This error occurs if you attempt to assign something other than a function to the compareWith property. For more information on proper usage of compareWith see the Angular forms documentation).
