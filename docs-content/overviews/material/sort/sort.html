matSort 和 mat-sort-header 用於為表格型資料新增排序狀態並顯示出來。

The matSort and mat-sort-header are used, respectively, to add sorting state and display to tabular data.

為表頭新增排序功能

Adding sort to table headers

要想為表頭新增排序行為和樣式，請把 <mat-sort-header> 元件新增到每個表頭中，並提供一個 id 來標識它。 這些表頭應該包含在一個帶有 matSort 指令的父元素中，當用戶在表頭上觸發排序時，該指令將會發出一個 matSortChange 事件。

To add sorting behavior and styling to a set of table headers, add the <mat-sort-header> component to each header and provide an id that will identify it. These headers should be contained within a parent element with the matSort directive, which will emit a matSortChange event when the user triggers sorting on the header.

使用者可以透過滑鼠點選或鍵盤動作來為表頭觸發排序。這時，matSort 就會發出一個 matSortChange 事件，其中包含觸發排序的表頭和排序的方向（asc 或 desc）。

Users can trigger the sort header through a mouse click or keyboard action. When this happens, the matSort will emit a matSortChange event that contains the ID of the header triggered and the direction to sort (asc or desc).

修改排序方向

Changing the sort order

預設情況下，排序表頭的排序方向先從 asc 開始，再點就變成 desc，再點一次則會移除排序。

By default, a sort header starts its sorting at asc and then desc. Triggering the sort header after desc will remove sorting.

要想逆轉所有表頭的排序順序，請把 matSort 指令的 matSortStart 屬性設定為 desc。 如果要對某個特定的表頭逆轉排序順序，請設定那個表頭自身的輸入屬性 start。

To reverse the sort order for all headers, set the matSortStart to desc on the matSort directive. To reverse the order only for a specific header, set the start input only on the header instead.

要想阻止該使用者從已經排序的列上清除排序狀態，請把 matSort 的 matSortDisableClear 屬性設定為 true 來影響所有表頭，如果只想針對特定的表頭，請把該表頭的 disableClear 設定為 true。

To prevent the user from clearing the sort state from an already sorted column, set matSortDisableClear to true on the matSort to affect all headers, or set disableClear to true on a specific header.

停用排序

Disabling sorting

如果你要阻止使用者修改所有列的排序狀態，可以繫結 mat-sort 的 matSortDisabled 屬性；如果只想針對單個 mat-sort-header，請繫結它的 disabled 屬性。

If you want to prevent the user from changing the sorting order of any column, you can use the matSortDisabled binding on the mat-sort, or the disabled on a single mat-sort-header.

在 mat-table 上使用排序

Using sort with the mat-table

當使用 mat-table 中的表頭時，不需要為它設定 mat-sort-header，因為預設情況下它將會使用列本身的 id。

When used on a mat-table header, it is not required to set a mat-sort-header id on because by default it will use the id of the column.

無障礙性

Accessibility

當你將 MatSortHeader 應用於標題單元格元素時，該組件會將標題單元格的內容包裝在按鈕內。然後標題單元格的文字內容就會成為排序按鈕的無障礙標籤。但是，標題單元格文字通常會描述列，而不會用來與控制元件互動執行排序操作。要清楚地表達標題如何執行排序，請始終使用輸入屬性 sortActionDescription 來提供按鈕元素的描述，例如『按姓氏排序』。

When you apply MatSortHeader to a header cell element, the component wraps the content of the header cell inside a button. The text content of the header cell then becomes the accessible label for the sort button. However, the header cell text typically describes the column and does not indicate that interacting with the control performs a sorting action. To clearly communicate that the header performs sorting, always use the sortActionDescription input to provide a description for the button element, such as "Sort by last name".

MatSortHeader 應用 aria-sort 屬性將當前排序狀態傳達給輔助技術。但是，大多數螢幕閱讀器不會播報 aria-sort 值的更改，這意味著螢幕閱讀器使用者不會收到排序發生的反饋。要解決此問題，請使用 MatSort 指令上的 matSortChange 事件透過來自 @angular/cdk/a11y 的 LiveAnnouncer 服務播報本次狀態更新。

MatSortHeader applies the aria-sort attribute to communicate the active sort state to assistive technology. However, most screen readers do not announce changes to the value of aria-sort, meaning that screen reader users do not receive feedback that sorting occurred. To remedy this, use the matSortChange event on the MatSort directive to announce state updates with the LiveAnnouncer service from @angular/cdk/a11y.

如果你的應用程式包含許多表和排序標頭，請考慮建立自定義指令以一致地應用 sortActionDescription 並播報排序狀態更改。

If your application contains many tables and sort headers, consider creating a custom directives to consistently apply sortActionDescription and announce sort state changes.
