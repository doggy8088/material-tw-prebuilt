Angular Material 提示框提供了一種文字標籤，當用戶的滑鼠懸停或長按某個元素時，它就會顯示出來。

The Angular Material tooltip provides a text label that is displayed when the user hovers over or longpresses an element.

定位

Positioning

提示框將會顯示在本元素的下方，但可以使用輸入屬性 matTooltipPosition 進行設定。 提示框可以顯示在元素的上方、下方、左側或右側，預設是下方。如果希望該提示框在 RTL 佈局下自動調換左右位置，則應該用 before 和 after 代替 left 和 right。

The tooltip will be displayed below the element but this can be configured using the matTooltipPosition input. The tooltip can be displayed above, below, left, or right of the element. By default the position will be below. If the tooltip should switch left/right positions in an RTL layout direction, then the positions before and after should be used instead of left and right, respectively.

位置Position 說明Description
above 總是顯示在元素上方Always display above the element
below 總是顯示在元素下方Always display beneath the element
left 總是顯示在元素左側Always display to the left of the element
right 總是顯示在元素右側Always display to the right of the element
before 在 LTR 佈局下顯示在左側，RTL 佈局下顯示在右側Display to the left in left-to-right layout and to the right in right-to-left layout
after 在 LTR 佈局下顯示在右側，RTL 佈局下顯示在左側Display to the right in left-to-right layout and to the left in right-to-left layout

根據顯示工具提示的位置， .mat-tooltip-panel 元素將接收可用於樣式的 CSS 類（例如新增箭頭）。可能的類是 mat-tooltip-panel-above 、 mat-tooltip-panel-below 、 mat-tooltip-panel-left 、 mat-tooltip-panel-right 。

Based on the position in which the tooltip is shown, the .mat-tooltip-panel element will receive a CSS class that can be used for style (e.g. to add an arrow). The possible classes are mat-tooltip-panel-above, mat-tooltip-panel-below, mat-tooltip-panel-left, mat-tooltip-panel-right.

要顯示相對於觸發它的滑鼠或觸控的工具提示，請使用 matTooltipPositionAtOrigin 輸入屬性。啟用此設定後，工具提示將相對於觸發器的原點而不是宿主元素顯示。在工具提示不是由觸控事件或滑鼠單擊觸發的情況下，它的顯示方式與此設定關閉時的顯示方式相同。

To display the tooltip relative to the mouse or touch that triggered it, use the matTooltipPositionAtOrigin input. With this setting turned on, the tooltip will display relative to the origin of the trigger rather than the host element. In cases where the tooltip is not triggered by a touch event or mouse click, it will display the same as if this setting was turned off.

顯示與隱藏

Showing and hiding

預設情況下，當用戶把滑鼠移到到觸發器元素上時，提示框會立刻顯示出來；滑鼠離開時會立刻隱藏。

By default, the tooltip will be immediately shown when the user's mouse hovers over the tooltip's trigger element and immediately hides when the user's mouse leaves.

在行動裝置上，當用戶 "長按" 觸發器元素時會顯示提示框，延遲 1500ms 後自動隱藏。

On mobile, the tooltip is displayed when the user longpresses the element and hides after a delay of 1500ms.

延遲顯示與隱藏

Show and hide delays

要想在顯示或隱藏提示框之前新增延遲，可以使用輸入屬性 matTooltipShowDelay 和 matTooltipHideDelay 來以毫秒為單位指定延遲時間。

To add a delay before showing or hiding the tooltip, you can use the inputs matTooltipShowDelay and matTooltipHideDelay to provide a delay time in milliseconds.

修改預設的延遲行為

Changing the default delay behavior

你可以使用依賴注入令牌 MAT_TOOLTIP_DEFAULT_OPTIONS 提供一個選項，來設定應用中提示框的預設顯示/隱藏延遲。

You can configure your app's tooltip default show/hide delays by configuring and providing your options using the MAT_TOOLTIP_DEFAULT_OPTIONS injection token.

手動呼叫 show() 和 hide()

Manually calling show() and hide()

要想手動顯示或隱藏提示框，你可以呼叫該指令的 show 和 hide 方法，它們可以接受一個以毫秒為單位的數字作為延遲時間。

To manually cause the tooltip to show or hide, you can call the show and hide directive methods, which both accept a number in milliseconds to delay before applying the display change.

禁止顯示提示框

Disabling the tooltip from showing

要完全停用提示框，請設定 matTooltipDisabled。停用之後，則永遠不會顯示出提示框。

To completely disable a tooltip, set matTooltipDisabled. While disabled, a tooltip will never be shown.

無障礙性

Accessibility

MatTooltip 會新增一個 aria-describedby 描述，它指向一個包含提示訊息的不可見元素。 當最終使用者把焦點移到用於觸發該提示框的元素上時，螢幕閱讀器就會讀出該提示訊息的內容。 aria-describedby 所指向的那個元素並不是提示框本身，而是該提示框內容的一個不可見副本，但它會始終存在於 DOM 中。

MatTooltip adds an aria-describedby description that provides a reference to a visually hidden element containing the tooltip's message. This provides screen-readers the information needed to read out the tooltip's contents when the end-user focuses on tooltip's trigger. The element referenced by aria-describedby is not the tooltip itself, but instead an invisible copy of the tooltip content that is always present in the DOM.

避免只顯示帶有滑鼠指標事件（如單擊和滑鼠輸入）的工具提示的互動。始終確保鍵盤使用者可以執行滑鼠和觸控使用者可用的相同操作集。

Avoid interactions that exclusively show a tooltip with pointer events like click and mouseenter. Always ensure that keyboard users can perform the same set of actions available to mouse and touch users.
